# Floorplan Phase 1 — 室內配置驗證版

> 規格定案日：2026-04-29  
> 目標完成：2026-06-03（5 週）  
> Branch：`feat/floorplan-mvp`（尚未開）  
> Route：`/floorplan` 或 `/layout`（待定）

## 1. 為什麼做這個

目標客群是**室內設計師**。他們手上已有 AutoCAD / SketchUp / Vectorworks，要他們換工具必須有一個那些做不到的事——

**護城河**：點平面圖上的櫃子 → 跳到 designer 改尺寸/榫卯/材料 → 回來自動更新配置 + 出材料單給師傅。  
這個閉環 SketchUp 跟 AutoCAD 都做不到。

Phase 1 的目的不是做完整 CAD（那是 Phase 2/3，11-15 週的事），而是**5 週後拿給 5-10 位室內設計師驗證會不會付錢**。如果驗證失敗就停。

## 2. 範圍

### 做

- **房間**：純矩形，輸入長 × 寬即可。背景顯示刻度與牆長標籤。
- **家具配置**：
  - 左側 Sidebar：「我的櫃子」清單，讀使用者在 designer 已存的設計
  - 拖到房間 → 顯示俯視圖 outline + 尺寸標籤
  - 旋轉 90°、自由拖、snap 到牆邊
  - 重疊偵測（紅框警告，不阻擋）
- **Designer 回路（核心賣點）**：
  - 雙擊櫃子 → modal 或新 tab 開 designer 編輯該設計
  - 編輯儲存 → 回配置頁 → 該櫃子尺寸自動更新
- **通用家具 icon（20 個最小集）**：
  床、沙發 ×2、餐桌、椅子、馬桶、洗手台、淋浴間、冰箱、洗衣機、電視、書桌、衣櫃、書櫃、邊桌、地毯、植栽、燈具、玄關鞋櫃、和室桌
  - 純 2D 線稿 SVG，**不可編輯參數**（佔位用）
- **輸出**：
  - A3 配置圖（房間 + 所有家具俯視圖 + 尺寸）
  - 材料單彙整：只彙整木頭仁設計的櫃子 BOM，一般家具不算
  - 列印 PDF

### 不做（Phase 2 才做）

- 多房間 / L 型房間 / 任意多邊形
- 牆 / 門 / 窗繪製
- 開門弧線、插座符號
- 尺寸鍊（dimension chain）、面積計算
- 自動立面圖（這個最難，3-4 週）
- 雲端設計檔存檔（先用 localStorage）

## 3. 技術決策

| 項目 | 決策 |
|---|---|
| Repo | wooden-ren-designer 同 repo |
| Route | `/floorplan` 或 `/layout`（動工時定） |
| Branch | `feat/floorplan-mvp` |
| 繪圖引擎 | 傾向 react-konva（拖放/旋轉/snap 成熟），動工時確認 designer 既有用法後最終定案 |
| 狀態管理 | URL state + localStorage |
| 方案門檻 | 「專業版」沿用既有 `PrintAccessGate` |
| 雲端存檔 | Phase 1 不做 |

## 5. 驗證指標（W5 結束後）

找 5-10 位室內設計師問：
1. 你現在用什麼工具做這件事？
2. 「點櫃子直接編輯尺寸」對你有用嗎？
3. 一個月 NT$ 多少你會訂閱？

依結果決定是否啟動 Phase 2（真平面圖 / 牆門窗 / 立面）。
