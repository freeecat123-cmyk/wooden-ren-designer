# drafting-math.md 對齐檢查 — 零件圖（Part Shop Drawing）

## 概述

本報告對標「零件圖」feature 需求與 `docs/drafting-math.md` (3760 行) 的涵蓋程度。零件圖目標：
- 單件局部坐標系下的正投影三視圖（長軸 = X 水平）
- 窮舉尺寸標註（整體 + 每個榫卯特徵）
- 榫卯特徵的引線與深度標記
- ×N 合併規則、安裝提示徽章、局部放大圖

## 不直接適用的 Section

| § | 為什麼不用 |
|---|----------|
| §B（榫卯細節圖） | 細節圖是**兩件間**的接合面特寫；零件圖是**單件**的視圖標註。結構平行但用途不同。後續可共用 SVG path 生成邏輯，但標題 / 尺寸策略要分開 |
| §G（擴充榫卯類型） | 粽角榫、霸王棖、銀錠榫等——描述榫卯**設計**，不是「榫卯在三視圖上怎麼標」。零件圖會參考但不強依賴 |
| §H（爆炸圖） | 多件組合視圖；零件圖是單件 |
| §I（自動標註） | 側面覆蓋相似（rbush 碰撞、短內長外堆疊），但零件圖強調「零件周邊標註」而非「全圖表格化標註」。可參考但要改架構 |
| §J（板材展開） | 三維料理，零件圖還是投影 2D |
| §L（木紋方向） | 三視圖不畫木紋（§L 的細節圖才畫），零件圖也不畫 |

## 缺失 — 新增 Section 建議

### **§A12（提議）：零件圖（Single-Part Shop Drawing）規範**
**內容應涵蓋**：
- **局部坐標框**設定（length = X 水平、thickness = Y 垂直、width = Z 深）
- **視圖標題格式**：「零件名 (×N) — 比例 1:2」例如「腿 ×4 — 比例 1:5」
- **×N 合併規則**：相同零件疊列時的標題、圖框調整
- **安裝提示徽章**（install hint）：小圖示顯示該零件在組裝件中的位置/方向
- **title block**：零件編號、材料、修改歷史（或簡化版的裁切清單連結）
- **坐標原點慣例**：底部中心 vs 其他（跟 butt-joint origin 對齐）
- **端面標記**（端面編號 A/B/C/D，對應三視圖投影）

**預計 500–800 字**。

---

### **§A6.5（提議）：榫卯特徵標註（Joinery Feature Callout）— 零件圖版**
**內容應涵蓋**：
- **Mortise 虛線框**投影規則（遠側面 + 可見性判定，複用 §A4）
- **Deep callout 符號**：`↧` 符號 / `THRU` 文字 / 盲榫 `GL` (go-line, 深 = ?mm)
- **Leader line 樣式**（§I4 的延伸）：從 mortise 邊框 → 數字標籤（深、寬、距邊）
- **距邊基準線** (distance-from-edge baseline)：選擇「上邊」或「左邊」做為量測起點，虛線 baseline + 垂直引線
- **多個 mortise 標註排列**：避免引線交叉；底部/側邊 baseline 優先；同軸 mortise 共線標註
- **特殊標記**：edge-protection 偏移、通榫 vs 盲榫視覺區分、tapered 零件上 mortise 位置（變動高度）
- **與 §B8 對照**：細節圖標 (t, L, s)；零件圖額外標 (距邊, face ID, splay 補償)

**預計 1000–1200 字**。

---

### **§A6.6（提議）：特徵尺寸清單（零件圖的窮舉策略）**
**內容應涵蓋**：
- **必標尺寸 tier 分級**：
  - **Tier 1（絕對必要）**：長 L、寬 W、厚 T（三個主自由度）
  - **Tier 2（榫卯相關）**：每個 mortise 的 (depth, width, from-edge)；every tenon 的 (length, width, shoulder)
  - **Tier 3（可選細節）**：圓角半徑、錐度、腳位 offset、鉸鏈孔位中心等
- **演繹不標的**：如果 depth = 對方厚度全長或 2/3，可標 `THRU` / `GL` 免數字；對稱零件對稱尺寸只標一側
- **冗餘檢測**：dimA + dimB ≈ dimC 時，保留 A + B，C 標 `(REF)` 或刪除
- **排序優先序**：從左上→右下→下方 baseline（配 §I3 堆疊邏輯）
- **參照 vs 實標**：(REF) 代表「推導值」，不傳給工廠作為公差基準

**預計 700–900 字**。

---

### **§A8.1（提議）：零件圖的局部細節放大（Detail Callout）**
**內容應涵蓋**：
- **細節選擇準則**：mortise 邊縁保護（±5mm 偏移）、tight tenon fit、複雜榫卯（粽角、走馬銷）、曲線過渡
- **引線和圓圈標籤**：複用 §A8 的「圓圈 + 字母」，但在零件圖脈絡下（不是多件的特寫）
- **放大比例常用值**：2:1（常用）、5:1（緊湊特徵）、10:1（微細 edge-protection）
- **細節框位置**：優先放在零件圖下方或右上（避免擋主要視圖）
- **與細節圖的區別**：零件圖細節是「局部視圖」（仍是 orthographic），細節圖是「兩件接合展開」（§B 型）

**預計 500–700 字**。

## 文檔 vs 需求的衝突 / 決策點

### ❌ 無衝突點
目前 §A～C 與零件圖需求**完全一致**，没有互相否定的規則。

### ⚠️ 需澄清的設計決策（可選）

**Q1：零件圖是否也遵循第三角法投影符號？**
- 文檔（§F5）：CNS 強制第三角法符號在三視圖右下角
- 實務（§D2）：已確認「客戶看不懂」→ 跳過
- **建議**：零件圖也跳過，保持簡潔。

**Q2：Mortise 表現方式（虛線框 vs 剖面線）**
- 文檔（§B6）：細節圖用虛線 + 剖面線表示 mortise 深度
- 零件圖考量：三視圖已用虛線表示「遠側面」，mortise 虛線如何區分？
- **建議**：
  - 視圖內 mortise 用細虛線 (stroke-dasharray="2 2")
  - 剖面線不需要（2D 視圖不切面）
  - 靠 leader + depth callout 補充深度資訊

**Q3：座標原點（origin）的一致性**
- 文檔（§A10.9）：Mortise origin = CENTER（part-local，X/Z 從中心量、Y 從底量）
- 零件圖場景：origin 應跟 visible geometry 一致（底部中心），不新增亂度
- **建議**：沿用 §A10.9 慣例，零件圖直接套

**Q4：Tapered 零件的 mortise 標註**
- 文檔（§A11）：Tapered 腳的 cross-section 隨 Y 變化 → apron 長度要補償
- 零件圖：Zero件本身是 tapered 腳時，mortise 位置會隨高度變化
- **建議**：在 §A6.5 的「距邊 baseline」加註：tapered 零件標註「下邊距」或「中心高距」，並註明「tapered 補償已內含」
