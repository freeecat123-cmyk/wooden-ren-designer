# 零件圖：匠師視角的必要資訊

> 真的拿圖去做的工匠在意什麼？資訊密度越高的圖越爛——下了料找不到重點。
> 以下挑 wrd 目錄裡 7 種「最會做壞」的零件，列出每張圖**非有不可**的東西。

---

## Top hard parts and what their drawings need

### 1. 圈椅椅圈五段 (`circle-chair`: `arm-rail-back` / `arm-rail-mid-l/r` / `arm-rail-side-l/r`)
**難在哪：**
- 五段都不是直料，俯視繞 Y 軸三個不同角度（後段 0°、中桿 ±45°、左右桿 ±90°），每段還有 `arch-bent` 的外凸弧度（26 / 14 / 28 mm）
- 五段端對端要互相吃進 `JOINT_OVERLAP = 32mm`，看 3D 圖判斷不出來「應該長 backLen 還是 backLen - 64」
- 楔釘榫（P2/P3 之後才會做）是椅圈的命脈——一錯整圈合不攏

**圖上必須有：**
- **每段的「弦長 + 弧高」**（不是只給弧長，工匠用弦+矢去畫弧線放樣）
- **兩端角度**：俯視標每段端面跟下一段端面的夾角（135° / 90°）
- **木紋走向**：箭頭沿弧線切向（不是順 length 軸——長料順紋彎弧會爆裂）
- **接點記號**：J_backmid / J_midside 在椅圈上的位置，連同「鄰段 ID」（"接 arm-rail-mid-r 的左端"）
- **截面寬從何處量起**：`sectionScale` 只乘 width，不乘 thickness——圖上要寫死 `W_BACK / W_MID / W_SIDE × scale` 各自的值
- 楔釘榫孔位（俯視透視一個小爆炸圖）

**常犯錯：** 把椅圈五段當直料下，做出來合圍少了 ~64mm × 4 = 256mm，整圈兜不上
**大師會加：** 俯視主圖旁邊放一張「中心線弧度套版（1:1 真寸）」，下料時直接套樣板鋸——這比任何尺寸都快

---

### 2. 馬蹄腳 (`shape: hoof`，明清案桌、`chinese-cabinet` `inward-hoof / outward-hoof`)
**難在哪：**
- 預設 `hoofMm = 90mm`（占 450mm 腳的 20%），這段腳趾朝家具外側翻起（`dirX / dirZ` 決定方向）
- 內翻馬蹄（明）vs 外翻馬蹄（清）方向完全不同，新手把方向做反整批報廢
- 腳趾段是 `hoofScale = 1.4` 放大——從直料下料時要預留比腳身寬 40% 的料

**圖上必須有：**
- **「腳趾朝哪一面」明確標示**：用箭頭+「外側朝廳堂前方」之類人話，不要只寫 `dirX = +1`
- **腳趾轉折線的 Y 高度**（從腳底量起的 `hoofMm`）
- **腳趾最寬處的截面尺寸**（`legSize × hoofScale`）
- **正視 + 側視兩張**（馬蹄是 3D 翻起，單視圖看不出腳趾長什麼樣）
- **下料建議**：腳身段 + 腳趾段一木做成，腳料厚至少要 `legSize × hoofScale`（畫圖直接寫毛料建議厚度）

**常犯錯：**
- 方向做反（內翻做成外翻）→ 整把椅子報廢
- 用腳身尺寸下料 → 鑿到一半發現腳趾段料不夠
- 把馬蹄當「腳底凸起的小腳座」做（誤解形狀）

**大師會加：** 一張「腳趾側視大樣（1:2）」+「腳趾與地面接觸線」的關係——用來判斷腳趾要不要刨地腳磨損面

---

### 3. 車旋花瓶腳 (`shape: lathe-turned`，`coat-rack` column、`round-table` pedestal)
**難在哪：**
- `lathe-turned` 是車床旋出來的，輪廓是一條 silhouette 曲線（多段半徑變化），不是單一直徑
- `coat-rack` 的 column 上面還要打 6-8 個 Shaker peg 的徑向榫眼，每個角度不同
- 木紋必須順 length（軸向），這部分輪廓不會錯，但端面接合面要平、要垂直軸

**圖上必須有：**
- **完整 silhouette 半剖面圖**：沿軸線剖一半，標出每個轉折點的 (Y 位置, 半徑)，列表也行
- **最大徑、最小徑、總長**：放大字級，車工放料時第一個要對的就是這三個
- **徑向榫眼的展開圖**：把圓柱展開成矩形，標出每個榫眼的 (Y, 旋轉角 θ, 深度)——這比在圓柱上畫榫眼清楚一百倍
- **木紋箭頭沿軸向**（畫一下，但每張車旋件都要有，免得有人忘了）
- **車削順序提示**：「先車整體大徑 → 分段定位環 → 最後修細頸」（可選，但對 hand-lathe 工匠有用）

**常犯錯：**
- 看到圖上 `轉折點 5: 半徑 18mm` 就直接卡尺量，沒留車削餘量（標 R18 應該寫「成品 R18，車削餘量 +1.5mm」）
- 徑向榫眼角度測錯（在圓柱上量角度極難）→ Shaker peg 全部歪

**大師會加：** 在 silhouette 旁加 1:1 樣板輪廓線（可印出來貼車床擋板用），徑向榫眼展開圖加「分度盤角度」（30°/60°/90° 表示用 6 分/3 分度法）

---

### 4. 外斜錐料牙條 (`shape: apron-trapezoid`，`bar-stool` / `dining-chair` / `bench` / `desk`)
**難在哪：**
- 牙條本身是梯形——`topLengthScale` 跟 `bottomLengthScale` 不一樣（補償外斜腳的 splay）
- 還可能帶 `bevelAngle` 半斜（`bevelMode: "half"`），只有上面那條邊斜，下面那條邊平
- 兩端的肩線跟外斜腳的肩線要對齊——牙條斜、腳也斜，端面是個雙重斜角

**圖上必須有：**
- **「上邊長 / 下邊長」雙標**（不是 length + scale，工匠看 scale 要心算很煩）
- **端面斜角度 θ**（從 90° 量起的偏角，搭桌鋸或斜切鋸用）
- **bevel 那條邊放大樣**：寫清楚「上邊倒 X° 斜面（半斜，靠座面側）」+ 哪面朝外
- **配的腳 ID 提示**：「左端接 leg-1（前左腳）下榫」「右端接 leg-2」——對位用
- **木紋順 length（長邊方向）**——這毫無懸念但還是要畫箭頭
- **加工順序**：「先按 top length 直料下 → 斜切兩端 → 後鑿榫頭」（避免新手先鑿榫再切端結果榫位偏掉）

**常犯錯：**
- 按長邊下料、忘了上下不等寬 → 牙條太短裝不進兩腳之間
- bevel 那條邊做反（朝外面而非朝座面）→ 牙條外露斜面破壞外觀
- 兩端的「複合斜角」用一般斜切鋸沒擋板做 → 角度偏 1-2°，牙條跟腳留 2mm 縫

**大師會加：** 端面斜切大樣（1:1）+ 鋸台斜角設定值（「桌鋸鋸片左傾 7°，擋板偏 5°」——直接搬到機器設定）

---

### 5. 圈椅聯幫棍 (`circle-chair`: `side-spindle-l/r`，`shape: splayed-round-tapered`)
**難在哪：**
- 圓棍 + 外斜（`dzMm = 38`，底面往 +Z 偏 38mm）+ 上下不等徑（`bottomScale = 1.1`，底略粗）
- 長度等於 `ringY - seatHeight`——椅圈到座框之間的對角距離，不是垂直高
- 上下兩端要分別對到座框跟椅圈，端面是斜的
- P1 簡化成 40×40 正方斷面圓棍是 wrd 內部現實——圖上仍要標「成品直徑 40，車旋」

**圖上必須有：**
- **斜置側視**：把聯幫棍橫放，標真實長度（hypot 後的值，不是垂直 spindleH）
- **兩端面與軸線的夾角**（座框端、椅圈端各一個 θ）
- **底徑、頂徑、車削總長**
- **配對 ID**：「下端入座板側框 seat-rail-left 的 mortise」「上端入 arm-rail-side-l 的 mortise」

**常犯錯：** 把長度當垂直高（`spindleH`）車料 → 短了 ~5mm，裝不進去

**大師會加：** 兩端面的「對位記號」（鋸座框邊起算）以及斜插時的工裝建議

---

### 6. 邊抹板心 框邊 (`chinese-cabinet`: `*-rail` + `*-side-panel`)
**難在哪：**
- 框邊是個小斷面長條，**外面要跟立柱齊平**（共面）但內面要開**槽**（容板心 5mm 深）
- 板心比內框小 10mm（兩邊各 5mm 嵌入槽），不能黏死——要讓木材吐縮
- 圓角櫃 splay 模式下，rail 跟 panel 都要跟立柱一起斜——`splayed-tapered` 的 `dxMm` 補底端外擴量
- rail 兩端都是 `shouldered-tenon`（帶肩榫）入立柱榫眼

**圖上必須有（rail）：**
- **斷面圖**：標出槽位置（離框邊內面 X mm，深 5mm，寬 = panelThickness + 1mm 餘量）
- **兩端肩榫尺寸**（hardcoded：`tenonLen / tenonW / tenonT`）
- **長度 = 跨距 + 兩端各 tenonLen**（這版 wrd butt-joint 慣例下要寫清楚 visible.length 是哪個量）
- **「外面 / 內面」標示**（內面開槽、外面齊平立柱）
- splay 圓角櫃：跟著立柱斜的角度（θ from horizontal）

**圖上必須有（panel）：**
- **板心是浮的，不上膠**——這行字一定要寫進圖，新手會習慣性塗膠把板心黏死
- **木紋方向**：板心是 frame-and-panel 的核心，板心一定**順 length 跑紋**（橫紋會把框撐裂）
- **上下抹槽嵌入量各 5mm**，所以板心毛料尺寸 = 內框可見尺寸 + 10mm
- 圓角櫃：頂寬 / 底寬雙標 + `panel taper ratio`

**常犯錯：**
- 板心上膠 → 季節變化板心撐爆框邊（明清古家具最致命錯誤）
- 板心按內框可見尺寸下料 → 嵌不進槽
- rail 槽開錯邊（外面開槽）→ 板心暴露在外

**大師會加：** 一張「框組裝順序」小圖：先鑿槽 → 後車榫頭，因為車完榫頭再鑿槽會崩榫；以及四角組合「先膠兩個 L 型半框，再合對」

---

### 7. 一木連做的後椅腿 (`dining-chair`: `leg-3/4` `isBackContinuous` 模式)
**難在哪：**
- 後腿從地板一路延伸到椅背頂端（`seatHeight + backHeight`），中間穿過座板高度
- 一根料同時當椅腿、背柱——座板要乾淨坐在腳上，又不能跟後腿穿模
- 牙板（apron）要從這根料的中段（座板下緣）入榫——榫眼在腿的「中段」不是端面
- 如果是 splayed 後腿，後傾的角度還要疊上去

**圖上必須有：**
- **總長 = `seatHeight + backHeight`**（不是 `legBaseHeight`）
- **座板下緣的位置記號**（Y = `seatHeight - seatThickness`）：這條線是分界，下面是腿、上面是背柱
- **中段榫眼大樣**：3 個榫眼可能集中在這一段——前後牙板 mortise + 側牙板 mortise + 椅背下橫檔 mortise，可能還疊著
- **頂端椅背的端面處理**：圓頭？方頭？倒角？這在 wrd 預設常被忽略
- **木紋必須順 length**——這對一木連做特別重要，橫紋下料會在座板高度斷掉

**常犯錯：**
- 把 `legBaseHeight` 當總長下料 → 沒有背柱可用
- 中段榫眼跟前腿榫眼用同一張圖共用 → 後腿榫眼比前腿少了「椅背入榫」，混淆
- 後腿做完才發現木紋方向偏 → 重買料

**大師會加：** 全長立面圖 + 三段截斷大樣（腳底段、座板穿過段、椅背段），每段標自己的截面工法

## Must-haves (drawing-system level)

每張零件圖都應該有的東西，沒有就算不完整：

1. **木紋方向箭頭**（三視圖每張都有）
   - 工匠看紋下料是第一動作。`grainDirection: "length"` 是內部慣例，圖上要畫箭頭+「順紋」二字
2. **「面向」記號**：哪面朝外 / 朝內 / 朝上 / 朝下（A 面 / B 面）
   - 非對稱件（馬蹄、apron 半斜、邊抹開槽面）做反就報廢
3. **安裝位置小角標**：右上角放 3D 預覽縮圖，紅圈圈出這個零件在整體位置
   - 圈椅 5 段椅圈長一樣外型，差在裝哪、別把後段裝成側桿
4. **配對 ID 提示**：榫頭旁註「配 leg-1 mortise-1」，mortise 旁註「容 apron-front tenon-1」
   - 對位+ wrd auto-merge 的時候用同一個編號系統
5. **尺寸只標到 1 位小數**（feedback `feedback_ui_number_precision.md` 慣例）
   - mm 標到整數最好，超過 1 位的工匠看不出意義
6. **「成品尺寸 vs 毛料尺寸」雙標**
   - 車旋件、刨光面、榫肩切削區一定要留餘量；圖上寫「車削後 ⌀18 / 毛料 ⌀22」
7. **榫頭 / 榫眼一律配 §B 細節圖編號**
   - drafting-math.md §B 已經整理好標準畫法，零件圖只放縮圖+引用 detail sheet 編號
8. **butt-joint 慣例宣告**
   - 圖框內小字「visible.length 為對接長度（含端面進入榫眼部分），實際裸露長度 = visible.length - 2 × tenonLen」——避免 wrd vs 傳統製圖兩種長度慣例打架

## Nice-to-haves

加分項，有就好用，沒有不致命：

1. **車旋件、弧線件：1:1 樣板輪廓**
   - 列印出來貼擋板/車床用
2. **加工順序建議**：「先鑿槽 → 後切榫」這類關鍵 sequence
3. **替代工法**：「無榫機時，手鑿馬腰榫的鑿線位置」（針對車旋徑向榫眼、馬蹄轉折這類精密件）
4. **鋸台/桌鋸設定值**：apron-trapezoid 端面斜角、bevel 半斜面，直接寫「桌鋸鋸片左傾 7°」
5. **每張圖右下角小資訊欄**：總材積 / 預估工時 / 木材編號（從 §X 報價系統拿）
6. **「跟誰一起切削」批量提示**：4 隻一樣的腿，圖上寫「批量切削 ×4，建議一次夾治具」

## Anti-patterns（不要做的事）

明確不要做、做了反而扣分的：

1. **不要把標註塞太密**
   - 三視圖塞滿尺寸線、leader line、文字，工匠拿起來找不到主尺寸
   - 規則：主視圖留三個關鍵尺寸（總長/總寬/總高），其他丟細節大樣
2. **不要把 `visible.length` 跟 `actual length` 混用同一個數字**
   - butt-joint 慣例對工程模型有意義，對工匠工作圖會誤導
   - 解法：圖上一律標「對接長度（含榫）」+「裸露長度」雙值，或選一種貼一致
3. **不要寫內部 shape kind 詞彙**（`apron-trapezoid` / `splayed-round-tapered` / `arch-bent`）
   - 翻成工匠術語：「梯形牙條」/「外斜圓錐棍」/「弧彎扶手桿」
4. **不要省略木紋方向**（即使 `grainDirection: "length"` 是預設）
   - 一張沒紋向箭頭的圖 = 開放式賭博
5. **不要把多個零件擠在同一張圖共用座標系**
   - 五段椅圈各畫一張，不要把五段擺在一張俯視圖上比例縮一半
6. **不要假設工匠看得懂 3D 立體投影**
   - 主圖永遠用三視正投影；3D 透視只放右上角小縮圖當定位用
7. **不要用「左 / 右 / 前 / 後」當絕對方向**（除非配上「腳底朝下、椅面朝 ↑」這種定位記號）
   - 工匠把料拿在手上是任意方向的——要用「外側面」「內側面」「鋸線方向」這類**料自己的**參考

---

## 結論建議給開發團隊

優先級：
1. **木紋方向 + 面向記號 + 配對 ID** → 這 3 件 80% 的零件圖都需要，先把這個架構搭好
2. **車旋件、弧形件、splayed/trapezoid 系列** → 這些用標準三視正投影不夠，要加「展開圖 / 1:1 樣板 / 端面大樣」
3. **butt-joint 慣例宣告**——這是 wrd 跟傳統製圖的最大歧義，圖框統一寫一行小字解釋
4. **drafting-math.md §B 榫卯細節圖**先做好，零件圖用引用編號的方式串接，避免每張圖重畫一次榫頭

別碰：
- 標到 0.01mm 精度（工匠笑出來）
- 一張圖塞八個視圖（看不到重點）
- 直接 export wrd 內部 shape kind 字串（工匠看不懂）
